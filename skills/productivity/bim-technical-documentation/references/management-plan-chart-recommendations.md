# Chart & Figure Recommendations by Management Plan Section

For museum fit-out / cultural projects with high integration complexity (AV, MEP, display cases, security, precision HVAC). Based on QC review of Section 13 "Interface & Coordination Management" for Aseer Regional Museum.

## Section 13 — Interface & Coordination Management (إدارة الواجهات والتنسيق)

### Why Charts Are Essential Here

Museum projects have the highest integration density in construction. AV/IT touches every trade. A missing data drop behind a showcase, an undersized AV room, or a cooling shortfall in a gallery kills the schedule. Text alone cannot manage this complexity — charts make the section actionable.

### Mandatory (must-have)

| # | Chart | Purpose |
|---|-------|---------|
| 1 | **Interface Matrix** — جدول ثنائي الأبعاد (طرف × طرف) | العمود الفقري للقسم. يوضح مَن يتعامل مع مَن. استخدم رموز: P = Physical, I = Informational, F = Functional. أطراف نموذجية: Samaya, CG, PMC, MEP Designer, AV Contractor, Showcase Specialist, Lighting Designer, Security Contractor, Rigging, Lifts |
| 2 | **RACI Matrix** — (نشاط/واجهة × طرف) مع R/A/C/I | يحسم الخلافات قبل وصولها للنزاع. لكل واجهة محددة: من Responsible, Accountable, Consulted, Informed. أمثلة على واجهات حرجة: "تزويد كهرباء لشاشات العرض التفاعلية", "فتحات التكييف في خزائن العرض", "كابل البيانات خلف الشاشات" |

### Strongly Recommended

| # | Chart | Purpose |
|---|-------|---------|
| 3 | **Communication Flowchart** — مسار تصعيد مشاكل الواجهات | يوضح دورة حياة مشكلة واجهة: Identification → Logging → Assignment → Resolution → Closure. مع تعدد المقاولين والاستشاريين، هذا يمنع الفوضى الإدارية |
| 4 | **Org Chart with Interface Points** — هيكل تنظيمي + نقاط التقاطع 🔗 | يوضح مسؤول التنسيق لكل واجهة. يستخدم رموز توضح نقاط الواجهات الرئيسية |
| 5 | **Timeline / Sequence Chart** — جدول زمني للتنسيق | يوضح تسلسل التنسيق بين التخصصات. في المتحف، بعض الأنظمة تسبق غيرها (MEP rough-in > تركيب خزائن العرض). يمنع إعادة العمل. شكل مقترح: Swimlane diagram أو Gantt مبسط |

### Optional (reference only)

| # | Chart | Note |
|---|-------|------|
| 6 | **Physical Interface Diagram** — مخطط يوضح نقاط الواجهات على مخطط المبنى | مهم تقنياً لكنه ينتمي لمخططات BIM/التصميم وليس خطة الإدارة. ذكره كمرجع مع رابط لمخططات BIM |

### General Principle

بدون هذي المخططات، القسم يكون وصف للطريق بدون خريطة. Interface Matrix + RACI هما الحد الأدنى المطلوب عشان القسم يكون قابل للتنفيذ.

## Workflow: How to Answer "Do We Need Charts Here?"

When the user asks whether charts/figures are needed in a specific section:

1. **Identify the section** — is it a standard PM section (Interface, Risk, Schedule, Comms, Quality)?
2. **Assess project complexity** — museum/cultural projects need more charts than standard builds due to AV/display/MEP/security integration
3. **Delegate to Kimi** — dispatch a QC sub-agent with context about the section number, project type, and complexity. Kimi returns structured recommendations per chart type with justification
4. **Present as priority table** — Mandatory / Strongly Recommended / Optional, with Arabic/English names and rationale
5. **Offer to create** — ask the user "تحتاج أساعدك في إنشاء أي من هذي المخططات؟"

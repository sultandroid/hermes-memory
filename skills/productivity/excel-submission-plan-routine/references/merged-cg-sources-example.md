# Merged CG Sources — CR Sheet Pattern

When multiple CG reviewers comment on the same submission package, merge into one CR sheet.

## Example: Aseer Arch Submission Plan

| Source | Date | Items | CR Prefix |
|--------|------|-------|-----------|
| Mohammad Elbaz | 27 Jun 2026 | 13 | CR-01 to CR-13 |
| Maged Zamzam | 16 Jun 2026 | 7 | CR-14 to CR-20 |

## CR Numbering Convention

- Architecture comments: CR-01 to CR-06
- Structure comments: CR-07 to CR-13
- Scenography package: CR-14 to CR-20
- Separate groups by source in the Summary sheet

## What Not to Merge

- **Structural BOD detailed comments** (14 comments from Abdrabo Shahin, 05 Jul, Code C) — these are a separate submission package (BOD + Loading Plans), not the same as the Submission Plan review. Track in Consultant Comment Register, not in the CR sheet.
- Two-stream rule: CR sheet = for CG submission. Consultant Comment Register = internal tracking. Never merge.

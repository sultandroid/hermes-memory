# Direct Work Request to Design Consultant (NRS)

## When to Use

You need to request new deliverables from the design consultant (NRS, ZNA, AD Engineering) — not forwarding CG review comments, but asking them to start a new batch or package of work (e.g., Batch 2 of 3D visuals, next package of drawings).

## Principles

1. **Short and direct.** One paragraph with the ask and a line of context. No verbose breakdown of every element — the consultant already has the drawing register.
2. **Reference programme dates.** Say "per the programme schedule" or "per our dates." Do NOT ask "when can you do this?" or "what timeline works?"
3. **Confirm reference material is ready.** Quick note like "stamped Stage 4 drawings on file" so they know they can proceed without waiting.
4. **Ask for their proposed start/timeline** — one line at the end. This is confirming, not negotiating.

## Template

```
Subject: [Package Name] – [Batch/Phase]

Dear Mr. Jim,

Hope you're well.

Following on from [previous batch/package], could you please proceed
with [next batch/package] per the programme schedule?

The batch covers the remaining [spaces/galleries/disciplines] on
[floors/areas]. All stamped [relevant stage] drawings are on file
for reference. Please confirm your start date and proposed timeline.

Best regards,

[Sender Name]
[Role]
Samaya Investment
```

## Example

```
Subject: 3D Visualization Package – Batch 2

Dear Mr. Jim,

Hope you're well.

Following on from Patch 1 (Basement visuals), could you please
proceed with Batch 2 of the 3D Visualization Package per the
programme schedule?

The batch covers the remaining galleries and spaces on the Lower
Ground, Ground, and First Floors. All stamped Stage 4 drawings
are on file for reference. Please confirm your start date and
proposed timeline.

Best regards,

Sultan Issa
Project Manager — Aseer Regional Museum
Samaya Investment
```

## Pitfalls

- **Don't list every space/gallery** in the email body — the consultant has the drawing register. Just name the floors.
- **Don't ask "what timeline works for you"** — reference the programme schedule. The dates are already set.
- **Don't explain the rationale** — they know what the deliverables are. Just state the ask.
- **Don't bundle with CG review comments** — if you just sent CG comments, send this as a separate email.

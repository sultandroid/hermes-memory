# Scope of Services (SOW) Document — CG Approval

## When to Use

Creating a formal Scope of Services document for a project team or subcontractor, submitted to CG for approval. Follows the same HTML template format as CV submittal packs but with different content sections.

## Structure

A SOW document typically has 5 pages:

| Page | Content |
|------|---------|
| **P01** | Cover page — logo strip, team title, project info, meta-grid, DC block (blank), QC sign-off |
| **P02** | Table of Contents |
| **P03** | Team Structure + Scope by person (part 1) |
| **P04** | Scope by person (part 2) + Deliverables matrix |
| **P05** | Reporting & Governance + Scope Exclusions |

## Content Sections

### Team Structure
- List each team member with: role, key qualifications (degree, years, certifications)
- Use cards/summary blocks with person name, role title, and brief credential line

### Scope of Services
- Group by person or by phase (Design-phase, Construction-phase, Advisory)
- Each person's scope bullets must match their **actual proposal/contract** — read the proposal document first
- Use active verbs: "Leads", "Provides", "Prepares", "Coordinates"
- Reference SoW clauses and ER Mostadam/SBC 1001 compliance

### Deliverables Matrix
Table format with columns: # | Deliverable | Responsible | Frequency
- List all expected outputs with clear owner and cadence

### Reporting & Governance
- Reporting line (all team → Project Director)
- Day-to-day coordination (through Technical Office)
- Submission platform (Aconex)
- Progress reporting cadence

### Scope Exclusions
- What the team does NOT do (e.g., Mostadam registration fees, authority fees, third-party testing, engineering calculations)

## Format Rules

Same as CV submittal packs:
- HTML only — no PDF generation
- DC fields (Doc No, Revision, Issue Date) blank
- Title: "Samaya Sustainability · Scope of Services"
- Logos strip on every page
- Carlito/Calibri font, A4 portrait, black/white formal style

## Verification
- [ ] All scope items trace to actual proposal/contract
- [ ] Exclusions clearly stated
- [ ] No DC metadata filled in
- [ ] Deliverables matrix complete with owners
- [ ] HTML source only — no PDF

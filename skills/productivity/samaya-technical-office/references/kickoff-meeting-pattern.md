# Key Personnel Kickoff Meeting — Pattern

## When to Use
When a new key personnel member is approved (Sustainability Manager, BIM Manager, QC Manager, etc.) and needs onboarding to the Aseer Museum project.

## Trigger
- CG approves a new key personnel appointment
- User says "prepare kickoff meeting with [name], discuss the scope"

## Deliverable
A 4-page A4-printable HTML document with:
1. Meeting info + 10-topic agenda + CG requirements status
2. Scope of Work table (12+ responsibilities with ER/SOW references)
3. Mostadam credit pathway table (target Silver 45+) + day-based milestones (D0→TOC+6m)
4. First 30-day task plan (10 tasks) + action items with owners

## How to Build

### 1. Gather Context Before Writing
- Pull from session history: previous scope discussions, CG approval emails, existing strategy docs
- Check CG's specific requirements from their approval response
- Check SoW §13.9 (for sustainability roles) or relevant contract clause
- Check RevC0x strategy document for current status

### 2. Document Structure (4 pages)
- **Header**: Samaya green strip with doc ref/rev/status/page
- **Cover block**: Meeting title, attendee list, duration, location
- **Page 1**: Objectives (6 numbered items), 10-topic agenda table, CG requirements status table
- **Page 2**: Scope of Work responsibility table (12 rows, columns: #, Responsibility, Reference, Phase, Deliverable)
- **Page 3**: Certification credit pathway (8 categories + total) + day-based milestones (D0→TOC+6m)
- **Page 4**: First 30-day task plan (10 tasks with priority tags) + document reference table + action items (7 items)

### 3. Color Palette
- Primary: #0F766E (dark green), #1E40AF (navy), #0F172A (dark navy)
- Tags: amber for pending/in-progress, red for urgent, green for approved/ongoing
- Background: #F0FDF4 for green-tinted notes, #F8FAFC for alt rows

### 4. Key Sections Always Include

#### Meeting Objectives
1. Induct into project structure/governance
2. Align on scope of work
3. Review current strategy/plan and agree timeline
4. Establish deliverables schedule and coordination protocol
5. Confirm stakeholder requirements
6. Assign first 30-day tasks

#### CG Requirements Table
| # | Requirement | Status | Action |
|---|---|---|---|
| Column 1: requirement description | Color-coded tag | Concrete next step |

#### Scope of Work Table
12+ responsibilities with columns: #, Responsibility, Reference (SoW/ER), Phase (Design/Construction), Deliverable

#### Certification Credit Pathway (for sustainability roles)
8 Mostadam categories (SC, EN, WT, MT, IEQ, CM, IN, AW) with available/target points, total with buffer row

#### First 30-Day Task Plan
10 tasks with Owner, Due, Priority (HIGH/MEDIUM/LOW color-coded)

### 5. Pre-Reading Pack for Attendee
Send these before the meeting:
- Current strategy/plan (e.g., RevC05)
- CG approval email
- SoW extract (relevant clauses)
- CV submittal pack
- R&R draft
- Ask them to prepare gap analysis before meeting

### 6. Post-Meeting Actions
- Issue final plan/strategy before CG's deadline
- Submit R&R for formal approval
- Set up weekly 30-min sync
- Establish coordination protocol with NRS/Lead Designer

## Pitfalls
- **CG deadline marches** — the 1-week submission deadline from CG approval is real. Kickoff must result in a clear path to meet it.
- **Don't assume Fida knows the project** — even though he was approved, he needs the full project context (10-month fit-out, not 36-month build, 870sqm, exhibition-specific Mostadam pathway)
- **All text must be in English** — kickoff document is for a bilingual audience but the working language for all deliverables is English.
- **Parties in attendance** — include PD, Tech Office Manager, QC (acting if needed), and the new appointee. Names only when confirmed.

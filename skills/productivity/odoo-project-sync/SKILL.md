# Odoo Project Data Sync

## When to use

- User asks to "map Odoo tasks to repo structure"
- User asks to "sync Odoo status to GitHub"
- User asks to "update status from Odoo automatically"
- User asks for "daily status report from Odoo"

## Architecture

```
Odoo Project (XML-RPC)
    │
    ├── fetch_all_tasks() → {id: {name, parent_id, stage_id, progress, deadline}}
    │
    ├── build_tree() → {parent_id: [child_id, ...]}
    │
    ├── parent_progress() → aggregate % from children
    │
    ├── generate_status() → markdown file with:
    │   • Snapshot (total/completed/in-progress/not-started)
    │   • Discipline Health (per-parent progress table)
    │   • Critical Active Issues (hardcoded issue→task mapping)
    │   • Recent Milestones (completed subtasks)
    │   • Deadlines (sorted, overdue flagged)
    │   • Next Actions (DD-stage 0% tasks with deadlines)
    │
    ├── git commit + push to GitHub
    │
    └── push_to_odoo() — optional: update Odoo progress from status changes
```

## Key patterns

### Fetch all tasks for a project
```python
tasks = models.execute_kw(DB, uid, PW, "project.task", "search_read",
    [[["project_id", "=", PROJECT_ID]],
     ["id", "name", "parent_id", "stage_id", "progress", "date_deadline", "user_ids"]],
    {"order": "parent_id asc, id asc", "limit": 500})
```

### Build parent-child tree
```python
tree = {}
for tid, t in tasks.items():
    pid = t["parent_id"]
    tree.setdefault(pid, []).append(tid)
```

### Calculate aggregate parent progress
```python
def parent_progress(tasks, tree, pid):
    kids = tree.get(pid, [])
    if not kids:
        return tasks[pid]["progress"]
    vals = [tasks[k]["progress"] for k in kids]
    return sum(vals) / len(vals) if vals else 0.0
```

### Discipline mapping pattern
```python
DISCIPLINES = {
    2938: ("01 — Architecture", "Architecture"),
    2940: ("03 — MEP & IT Engineering", "MEP"),
    # ... parent_task_id: (label, short_name)
}
```

### Critical issue → Odoo task mapping
```python
CRITICAL_ISSUE_TASKS = {
    "SI-CG-ASEER-007": [2983, 2980],
    "IFC-0004 Life Safety Code C": [2891, 3005, 3050],
    # ... issue_name: [task_id, ...]
}
```


## Cron setup

```bash
# 4× daily: 06:00, 10:00, 14:00, 18:00 KSA
hermes cron create \
  --name "Aseer PM Status" \
  --schedule "0 6,10,14,18 * * *" \
  --script aseer-pm-status.py \
  --workdir ~/aseer-museum-pm \
  --deliver origin \
  --toolsets terminal
```

## Pitfalls

- **limit=500** — if a project has >500 tasks, paginate with `offset`
- **Odoo progress is 0.0–1.0** (float), not percentage. Multiply by 100 for display.
- **date_deadline** can be `False` (no deadline set) — check before `.isoformat()`
- **parent_id** is `[id, name]` tuple when set, `False` when not — check with `t["parent_id"][0] if t["parent_id"] else 0`
- **stage_id** is `[id, name]` tuple — same pattern
- **user_ids** is a list of ints (user IDs), not tuples
- **OneDrive files** may hit "Resource deadlock avoided" — use `_Project_Memory/` subfolder as fallback
- **Git push conflicts** — the cron runs 4× daily; if PROJECT_MEMORY.md hasn't changed, skip commit to avoid empty commits
- **Telegram messages** — the script's stdout is the Telegram message. Keep it concise (bullet list of changes, not full file dump)

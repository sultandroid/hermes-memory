# Plan Document — Personnel & Sign-Off Audit

When reviewing a project management plan document (Resource Management Plan, DMP, PEP, HSE Plan, BEP, etc.) for stakeholder/personnel accuracy, apply this checklist.

## Pre-Audit: Load Known Team Data

Before reviewing the document, recall the current project team from memory / session_search:

- **Project Director** — current name, appointment date
- **BIM Manager**
- **Construction Manager**
- **Tech Office Manager**
- **QA/QC Manager** — note if VACANT
- **HSSE Manager**
- **Procurement**
- **Document Controller**
- **Project Engineer**
- **BIM/Doc Engineer**

## Audit Checklist

### 1. Org Chart / Leadership Tables

- [ ] Every named person matches current team (not departed staff)
- [ ] Titles/roles are correct for each person
- [ ] PD appointment note is included if newly appointed
- [ ] No "acting" labels on confirmed roles

### 2. Sign-Off Tables (usually in §1.4 and §10.6)

There are typically TWO sign-off tables — one near the front (Document QC) and one near the end (Document Sign-Off). Both must be updated.

| Row | Rule |
|-----|------|
| Prepared by | Usually the author — confirm correct |
| Checked by | BIM Manager or designee |
| Reviewed by (QA/QC) | **If vacant → add named acting signatory with `(acting)`** e.g. Construction Manager who can sign off on behalf of QA/QC |
| Approved by | Project Director |
| Issued by (Document Controller) | **Always fill in the DC name** — do not leave blank |

**Key correction the user enforces:** Never leave QA/QC *or* Document Controller blank. If QA/QC is vacant, the Construction Manager (or other authorized person) signs as acting. The DC name is known — always add it.

### 3. Risk Register Personnel References

- [ ] Check risk descriptions for stale language (e.g. "acting PD" when PD is confirmed)
- [ ] Check risk owners — are they current team members?
- [ ] Check impact statements — do they still reflect the current org?

### 4. Cross-Document Consistency

- [ ] Names in org chart match names in sign-off tables
- [ ] Abbreviations/spelling are consistent (e.g. Abdelhameed vs Abdelhamid — use the document's existing spelling)
- [ ] All Tier 1-3 roles referenced in the document align with current staffing

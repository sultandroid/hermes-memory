# DC Submission Transmittal Pattern

## When to use

When the Document Controller (Hesham) needs a formal cover document to submit a subcontractor prequalification package to CG via Aconex. The transmittal is the Aconex cover sheet — it lists all attachments and gives the reviewer context.

## Distinction from other documents

| Document | Audience | Purpose | Author |
|----------|----------|---------|--------|
| **DC Submission Transmittal** (this doc) | CG | Formal cover for Aconex submission — lists all attachments | Samaya DC |
| CG Support Letter | CG | Samaya recommends supplier for approval (1-page recommendation) | Tech Office |
| Prequalification Letter | Samaya | Supplier confirms project understanding (RACI + risk register) | Supplier (or Samaya on their behalf) |

## Document structure

| Section | Content |
|---------|---------|
| Header | Samaya Investment, Technical Office - BIM Unit, project name, doc ref (MOC-MUS-ASE-1K0-ZD-XXXX) |
| Title | "SUBMITTAL - PREQUALIFICATION OF [TRADE]" |
| TO | Consultancy Group (CG), ACE Moharram-Bakhoum (PMC), Ministry of Culture (MoC) - via CG |
| FROM | Samaya Investment - Technical Office |
| DATE | Issue date |
| SUBJECT | Supplier name - Prequalification Submission |
| Body 1 | Samaya submits [Supplier] for prequalification as [Trade] for the project |
| Body 2 | Company credentials summary (CR, years, MoC portfolio if any, team) |
| Body 3 | Scope documents reviewed and understood |
| Body 4 | Samaya recommends approval |
| Attachments table | #, Document, File Name, Status (4-5 items) |
| Sign-off | Prepared by / Reviewed by / Approved by + signature block |

## Attachments table

Typical 5-attachment pattern for a prequalification submission:

| # | Document | File Name | Status |
|---|----------|-----------|--------|
| 1 | Company Profile | [Supplier] Profile.pdf | Original |
| 2 | Signed Prequalification Letter | [Supplier]_Prequalification_Letter.pdf | Signed by supplier |
| 3 | CG Support Letter | [Supplier]_CG_Support_Letter.docx | Samaya cover |
| 4 | Prequalification Support Document | [Supplier]_Prequalification_Support.docx | Supporting info |
| 5 | Compliance Sheet | [Supplier]_Compliance_Sheet.xlsx | Samaya assessment |

## Generation approach

Use SamayaDoc template with project header/footer. Doc ref pattern: MOC-MUS-ASE-1K0-ZD-XXXX (ZD = general document). The attachments table uses 4 columns with col_widths_cm=[1.0, 5.0, 7.0, 3.5].

## Pitfalls

- The transmittal is NOT the CG Support Letter — do not replace the existing support letter with this document. The transmittal is the cover, the support letter is the recommendation.
- Route through DC (Hesham) for Aconex upload — the transmittal alone is not a submission, it needs to go through document control with all attachments.
- Date the transmittal the same day it is issued, not the day the supplier signed their PQ letter.
- If the supplier was prequalified before the compliance sheet was created, the compliance sheet can be a separate later submission.

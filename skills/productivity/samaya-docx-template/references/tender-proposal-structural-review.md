# Tender Proposal Structural Review Checklist

Use when asked to review a technical proposal PDF or DOCX for a tender submission. This is a structural/content review, not a pricing review.

## 14-Point Checklist

### Critical (must fix before submission)

| # | Check | What to look for |
|---|-------|------------------|
| 1 | **Executive Summary** | Does it open with a one-page snapshot? Project name, client, location, scope, key differentiators, key commitments (safety, quality, programme). If missing, flag as critical. |
| 2 | **Table of Contents** | Are page numbers populated? If all entries show "—" or "Error! Bookmark not defined", flag. |
| 3 | **Org Chart** | Does the org chart match the CV section? If the PM named in the org chart differs from the CVs, flag the mismatch. Is the org chart project-specific or a generic company structure? |
| 4 | **Risk Register** | Is there a proper risk register with 15+ risks, categories, probability/impact, mitigation, owner? If only 3 bullet points, flag. |
| 5 | **Document Control** | Is there a proper section covering EDMS, transmittal procedures, RFI workflow, submittal workflow, version control, distribution matrix? If only 2 bullet points, flag. |
| 6 | **Past Project Portfolio** | Does the proposal include a structured portfolio appendix (3+ projects >2,000 sqm with client, value, scope, duration)? If a company profile is appended instead, flag. |

### Medium (should fix)

| # | Check | What to look for |
|---|-------|------------------|
| 7 | **Scope Section** | Is it project-specific? Does it reference the actual galleries/spaces by name? Does it mention the designer? Or is it generic boilerplate? |
| 8 | **Methodology Diagrams** | Are there flowcharts or process diagrams for the execution methodology? Or is it text-only? |
| 9 | **HSE Plan** | Is there a project-specific HSE outline (zero-incident target, risk assessment, training, emergency response, dust/noise/vibration control)? Or does it reference generic standards? |
| 10 | **Quality Plan** | Is there a quality plan outline (ITP process, hold/witness points, mock-up approval, NCR process, commissioning protocol)? Or just ISO 9001 reference? |
| 11 | **Value Engineering** | Is there a dedicated VE section? Or is it missing entirely? |
| 12 | **Stakeholder Communication** | Is there a communication plan (meeting cadence, reporting structure, escalation matrix)? Or missing? |
| 13 | **Sustainability** | Is there a sustainability section (Mostadam/SBC 1001 compliance, waste management, local content)? Or missing? |
| 14 | **Warranty & After-Sales** | Is there a warranty section (DLP, manufacturer warranties, spare parts, maintenance support)? Or missing? |

## Area Discrepancy Check

When the proposal states an area figure, cross-reference against the SOW and pricing model:

- **SOW** often states net exhibition area (galleries only)
- **Pricing model** often states total project area (galleries + supporting)
- If the proposal says "3,500+ m² across 7 galleries" but the SOW says "~870+ sqm exhibition area", the proposal is misleading — fix to clarify the split

## Project Context Rule (CRITICAL)

When writing review comments or section-by-section feedback, **never cross-reference another project**. If the proposal is for RCRC, do not mention Aseer, Zamzam, or any other project in the commentary. A single wrong project reference undermines credibility. The user will correct: "this not aseer project we work in RCRC exibition only."

## Presentation

Present findings as a table with:
- Section name
- Status (✅ / ⚠️ / ❌)
- Brief finding
- Severity (Critical / Medium / Low)

Group by severity. End with an overall score (e.g. 6.5/10) and a one-line verdict.

# Contract Drafting Document Pack — Worked Example

## Context: ZNA Studio (Lighting Designer) — June 2026

When preparing to send a subcontractor their contract documents, the pack must include:

## Document Pack Contents

| File | Source | Notes |
|------|--------|-------|
| Scope of Work (Rev applicable) | Tech Office — submitted version | Use the version that was sent to CG, not internal later revisions |
| Design Management Plan | From email — NRS-signed or CG-stamped | **Never use internal Word draft** — always extract the signed/stamped PDF from Outlook. User will call this out. |
| CG response (Code B/C) | From email — CG reviewer's reply | So the sub sees the comments they need to address |
| Submittal Register | Tech Office | For payment milestone alignment — ensure milestones map to submittal stages (50/90/100% IFC), not the sub's internal progress percentages |
| Sub's own quotation (copy) | From email | Reference for cross-checking |

## DMP sourcing from email

Extract TWO versions:
1. **NRS-signed PDF** — the designer-signed/stamped copy (e.g., Jim Richards from Nissen Richards Studio)
2. **CG Code B response** — the stamped submittal form with CG reviewer's comments

Both go into `08_Contract_Drafting/`. The NRS-signed copy is the one to send the sub as the approved DMP. The CG response goes as reference so they see the comments.

## Payment schedule alignment

Subs often submit payment milestones based on their internal progress (e.g., Commencement → 50% → 75% → 100%). These must be realigned to the project's submittal register stages (50% Design → 90% Design → 100% Design → IFC/AFC) so payments track to actual approved submissions.

Request in the reply: "Payment milestones to align with our submittal register stages, not your internal percentage."

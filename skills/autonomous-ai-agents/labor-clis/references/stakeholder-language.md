# Stakeholder Language — Reporting Guidelines

When producing reports, tables, or analysis for the Samaya BIM Unit:

## Replace Jargon with Meaning

| Instead of | Write |
|---|---|
| EXTRACTED / PDF ONLY / REF ONLY | DEFINES NRS RESPONSIBILITIES / PAYMENT BASIS VERIFIED / BACKGROUND NOT CRITICAL |
| Status: DONE / NOT TRACKED | DELIVERED ✓ / ⚠️ AT RISK — no tracking |
| Partial / Future | ⚠️ PARTIALLY DONE — [cause] / NOT YET STARTED — [blocker] |
| 0% / N/A | Not started — [who is responsible for delay] |
| All received, 0 stamped | Complete — usable for next phase |

## Column Headers That Serve Stakeholders

- "Status" → **"Stakeholder Value"** or **"What It Means"**
- "Notes" → **"Action Needed"** or **"Risk"**
- "Remarks" → **"Decision Required"**

## Invoice Language

- **Don't say** "REJECT" — say **"HOLD"**
- **Don't say** "zero EV" without context — say **"partial EV (~XK) from [specific work done], HOLD balance"**
- **Recommendation format:** "Pay ~XK for confirmed work; HOLD ~XK balance"

## Report Scope

- NRS reports are **day-snapshots** of earned value vs payments paid
- Core content only: what's delivered, what's paid, EV calculation, variance, outstanding, next steps
- Strip: contract structure tables, resource schedules, penalty clauses, workflow maps, obligation tracking
- If user asks for more detail, add as reference appendix, not main body

## Delay Attribution

When a package is delayed, attribute responsibility clearly:
- "Delayed by Samaya — [reason]"
- "Delayed by Specialist — [reason]"
- "Delayed by SI007 — owner-directed change, not NRS fault"
- Never blame a contractor for delays caused by the client or by missing specialist inputs

# CG Reviewer Behavior Patterns — Aseer Regional Museum

> Condensed reference from 13 lessons learned. Use these patterns to predict CG behavior and avoid repeat rejections.

## Mansour Alrezeni — Code Enforcer

- **Will not accept spec deviations** even when manufacturer advises against the material
- Treats CG review as binary compliance check, not risk-based decision
- **Do not propose alternatives or splits** to him — he will reject
- Escalate structural decisions (track splitting, material substitution) to CG PM (Elbaz)
- Submit only what the spec says. Nothing more, nothing less.

## CG Conflates Deliverables

- Uses one submittal review as a gate to demand content from unrelated disciplines
- Example: Material Board (1G-0003) returned Code C because AV/IT/Lighting engineering schedules weren't included
- **When CG returns Code C with cross-discipline comments**, check if comments are in-scope before accepting them all
- Respond clarifying scope boundary; separate applicable comments from non-applicable ones

## CG Unaware of NRS-MoC Deliverables

- CG often requests design-stage content that was already delivered under the original NRS-MoC contract
- **Before responding to any CG request for design-stage content**, verify if it was already delivered under the original design contract
- Direct CG to existing documentation rather than re-delivering

## CG Reviewer Assignment Unpredictable

- CG does not publish reviewer assignments in advance
- Prepare every submission to the strictest reviewer's standard (Mansour/Eslam level)
- If you don't know the reviewer, assume the strictest possible review

## Common CG Rejection Patterns (ranked)

1. Missing supporting documentation (certificates, test reports, CVs, org charts)
2. Incomplete submission / missing required content (appendices, signatures)
3. Non-compliance with specifications / ER requirements
4. Missing specialist / consultant endorsement (NRS review, FLS consultant)
5. Sequencing / workflow violations (IFC before PDD)
6. Insufficient technical data (load calculations, SLDs, cable schedules)
7. Single-source supplier (CG requires 3 options for every material)
8. Cross-disciplinary coordination gaps
9. Organisational chart / team structure deficiencies
10. Disputed milestones / contractual interpretation differences

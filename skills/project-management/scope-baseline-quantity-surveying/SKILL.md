# Scope Baseline & Quantity Surveying (Tender Pricing)

## When to Use
User asks to build a quantity surveying baseline for a tender pricing study — quantify, classify, flag gaps — or any variant: "build quantity register", "scope baseline", "QS register", "pricing baseline", "discrepancy analysis".

## Core Methodology — 7 Registers

### A) Document Register
Register EVERY input document: code, title, revision, date, format. Group by type (design PDFs, Excel data, BoQ, RFQ drafts, RFI/site reports, analysis, CAD). Each document gets a unique code.

### B) Discrepancy Log
Cross-compare parallel documents. For each discrepancy:
- Type (missing items, naming inconsistency, area code conflict, range mismatch, drawing-only content)
- Severity (CRITICAL / HIGH / MEDIUM / LOW)
- Detail — exact codes, exact sources
- ACTION — recommended resolution
- Affected document references

Key checks to always perform:
1. Furniture PDF vs Excel completeness
2. Area-code key consistency across ALL documents (watch for single-letter conflicts like CF/CA/FL)
3. Material code range counts (CL/FL/WL across PDF vs Excel)
4. Drawing-only content that cannot be quantified from text
5. ELV/MEP ballpark estimates vs detailed design
6. HVAC scope presence/absence

### C) WBS Register
Structure: GROUP-ZONE-PACKAGE-SEQ. Groups = major scope divisions (e.g. SUP/EXH/GEN). Zones = rooms/spaces within each group. Packages = work packages per zone. Mark scope present Y/N for each line. Include items NOT in BoQ that are flagged as gaps.

### D) Quantity Register
For every line: code, description, quantity, unit, quantity-class (area | count | system), source_doc, source_ref, method (measured | scheduled | assumed).

System packages (AV, LED, scenic, models, interactives) keyed to scenographic codes. Seed quantities from RFQ drafts where confirmed. Equipment counts consolidated across galleries.

### E) Open Items / RFI List
Every unconfirmed quantity, every ambiguous scope item, every document extraction failure → logged with priority, source, gap description, recommended action, and deadline.

### F) Scope-Gap & Risk Register
Each risk: severity (HIGH/MEDIUM/LOW), probability (1-5), impact (1-5), P×I rating, description, pricing treatment, affected line codes, owning RFI. Never price a risk — state the treatment (exclude from base, add provisional sum, add contingency, qualify).

### G) RFI Register
Every RFI: subject, recipient, scope reference, priority, date, status, findings summary, linked risks, outstanding questions, next step. Include both existing RFIs from the package AND new RFIs generated by the analysis.

## Guardrails

1. **Never invent a quantity.** If a value is not in any document:
   - Set method=assumed
   - Add to Open Items list
   - Flag with source and basis for assumption

**OneDrive Deliverable Workflow (Mandatory for all Samaya/Tqanny projects)**
- Never write .xlsx/.md/.pdf deliverables directly to any OneDrive path (CloudStorage or Group Containers).
- Stage all artifacts to `/tmp/`.
- Copy using Finder AppleScript: `osascript -e 'tell application "Finder" to duplicate POSIX file "/tmp/..." to folder POSIX file "..." with replacing'`.
- Always verify the copied file starts with `PK\x03\x04` (`xxd -l 8 <path>`). If it does not, the file is corrupt/placeholder — delete and repeat the copy step.

2. **Never resolve a discrepancy silently.** Log it in the Discrepancy Log with a recommended action. If two documents conflict, state which one you're using as the pricing basis and why, then flag the other for confirmation.

3. **Quantities only.** No rates, no prices, no costs in the Quantity Register. The Risk Register may reference provisional sums as pricing treatments, but these are scope-level, not item-level prices.

4. **Every row has a source.** Quantity Register: source_doc + source_ref mandatory. Risk Register: owning RFI mandatory. Discrepancy: action mandatory.

5. **Area-code conflicts are CRITICAL.** A single-letter ambiguity (CF = Cafeteria vs Conference, CA = Café vs unknown) propagates across every material/finish/furniture line. Flag as CRITICAL, state the recommended resolution, and require BMA/client confirmation before take-off.

6. **MEP/ELV ballpark estimates are HIGH risk.** Items priced as "ballpark estimate" or "provisional sum" without design basis must be flagged as high-severity risks. Do not convert to firm rates without detailed design.

7. **HVAC scope gap is a mandatory check.** Every exhibition/museum fit-out has significant heat load from AV and LED walls. If the BoQ has zero air conditioning lines, flag as a HIGH-severity risk immediately.

## Integration with Samaya Workflows
- Deliverable goes to `_PRICING_DOCS/` as a markdown file with structured tables
- Companion Excel file for the full Quantity Register (197+ lines) may be generated separately
- Results feed into the pricing model (RCRC_PRICING_MODEL_Complete.xlsx)
- New RFIs generated by the analysis get added to the RFI register and issued

## Pitfalls
- **Truncated summaries:** Design study summaries may be truncated (e.g. Furniture Excel showing 7 of 30 rows). Always verify by reading the original Excel with openpyxl before declaring completeness.
- **Image-based PDFs:** Exhibition material schedules are often image-only. Cannot extract codes programmatically. Flag immediately.
- **Duplicate area codes in material lists:** Watch for data-entry errors like "CF / FL / SM / LB / ... / CF" — CF appears at both start and end.
- **Period vs dash in codes:** "S-BR.CH.01" vs "S-BR-CH.02" — minor but breaks string matching. Standardize early.
- **LS vs itemized furniture:** BoQ often has furniture as Lump Sum while RFQs itemize specific counts. Cross-reference and flag any mismatch.

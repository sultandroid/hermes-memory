## Template: Send DD Package to NRS for Stamping

Use when a Design Development (DD) submittal package is ready and needs NRS review and stamp before formal submission.

```
**To:** Nissen Richards Studio (NRS)
**Subject:** {Discipline} Package {NN} — Design Development Submission for Review & Stamp — Aseer Regional Museum

Dear NRS,

Please find attached {Discipline} Package {NN} for your review and stamping. This is the {sequence} design development (DD) submission for the {discipline} discipline.

**Package contents:**

| # | Item | Format |
|---|------|--------|
| 0 | {Submission Plan} | PDF + XLSX |
| 1 | {Design Base Report} | PDF |
|   | — {Supporting reports} | PDF |
| 2 | {Calculations/Reports} | PDF |
|   | — {Detail items} | PDF |
| 3 | {Drawing set 1} ({Drawing ref}) | PDF + DWG |
| 4 | {Drawing set 2} ({Drawing ref}) | PDF + DWG |
| 5 | {Drawing set 3} ({Drawing ref}) | PDF + DWG |

**Drawing reference:** {Drawing code pattern}
**Status:** For Design Development Review & Stamp

Please review and provide your stamped approval or comments at your earliest convenience.

Best regards,

**Eng. Mohamed Sultan Abbas**
Technical Office Manager — BIM Unit
Samaya Investment Company
```

### Key Rules for DD Package Emails

- **Package contents table**: List every deliverable in the package with format (PDF, DWG, XLSX)
- **Drawing reference**: Show the code pattern (e.g., `MOC-ASE-ME-MHV-{type}-BF-DDD`)
- **Status line**: Always `For Design Development Review & Stamp`
- **No CC list** — NRS handles distribution internally
- **No deadline** — NRS knows the programme; don't add pressure in the email

# Tender Technical Proposal — Checklist & Rules

## Do NOT include
- Any monetary values (USD, SAR, $, SR amounts) — pricing belongs in the commercial proposal
- Budget breakdowns per gallery or per scope item
- Total project value
- Unit prices or rates

## Do include
- Methodology and approach descriptions
- Team qualifications and experience
- Gallery/scope technical descriptions (equipment types, quantities without prices)
- Compliance statements against ER requirements
- Timeline and programme (without financial milestones)
- Similar project experience (without contract values)

## Section-level checklist

| Section | Must have | Must NOT have |
|---------|-----------|---------------|
| Executive Summary | Project understanding, team differentiators | Total project value |
| Project Understanding | Design appreciation, gallery descriptions | Gallery budget figures |
| Company Profile | History, services, group companies | Revenue, project values |
| Relevant Experience | Project names, locations, scope descriptions | Contract values, SAR amounts |
| Execution Methodology | RIBA stages, workflow diagrams, BIM approach | Cost of design phases |
| Team & Organization | Org chart, key personnel, roles | Salary rates, resource costs |
| Programme | Phases, duration, milestones | Cost per phase |
| Risk Register | Risk descriptions, mitigation strategies | Cost impact amounts, contingency values |

## Common pitfalls to check before delivery
1. Search for: `$`, `SAR`, `USD`, `مليون`, `ريال` — remove any monetary values
2. Verify all prices in gallery descriptions, experience tables, and KPI grids are stripped
3. The technical proposal should read as a pure technical document — if a number looks like a price, remove it
4. Replace "budget" references with "classification" or "scope" language

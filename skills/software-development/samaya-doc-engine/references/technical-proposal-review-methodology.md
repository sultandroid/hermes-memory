# Technical Proposal — Section-by-Section Review Methodology

Use this when the user asks you to review a received technical proposal PDF (from a subcontractor, competitor, or Samaya's own draft). Do NOT use for Samaya's own outgoing proposals (use the generation checklist instead).

## Review dimensions

Score each section on: Completeness, Accuracy, Consistency, Specificity, Professional Presentation, Compliance with ER.

## Section-by-section evaluation criteria

### Cover Page
- Client, Designer, Contractor correctly identified?
- Document reference, revision, date present?
- Bilingual if required?
- Exhibition/scope area stated?
- Submission status clear (For Tender / For Review)?

### Table of Contents
- Page numbers populated (not "—")?
- All listed sections actually exist in the body?
- Page count in header matches actual pages?
- Appendices listed and present?

### Executive Summary (if present)
- One-page snapshot of project, scope, differentiators?
- Key commitments (safety, quality, programme)?
- If missing → flag as Critical gap

### Introduction / Project Understanding
- Project-specific language (not generic boilerplate)?
- References the client, designer, and specific exhibition theme?
- Demonstrates understanding of site constraints (occupied facility, etc.)?

### Project Objectives
- Specific, measurable targets?
- Or generic statements that could apply to any project?

### Scope of Work
- Gallery-by-gallery breakdown (G1, G2, etc.)?
- References the designer (BMA, NRS, etc.)?
- Exhibition area stated and consistent with SOW?
- Exclusions and assumptions listed?
- Or generic 8-phase methodology with no project specifics?

### Project Organization
- Project-specific org chart (not company-wide structure)?
- PM name in org chart matches CV section?
- Responsibility Assignment Matrix (RAM) present?
- Key roles named with actual personnel?

### Resource Management
- Headcount numbers and resource histograms?
- Equipment list specific to project?
- Or generic categories?

### Procurement Strategy
- Long-lead item list?
- Procurement schedule?
- Local content commitment?

### Site Management & Logistics
- Site-specific constraints addressed (occupied facility, access, working hours)?
- Protection measures for existing building?

### HSE
- Project-specific HSE plan outline?
- Risk assessment methodology?
- Emergency response plan?
- Dust/noise/vibration control for occupied facility?
- Or 3 generic bullet points? → flag as Critical gap

### Quality Management
- ISO standard referenced?
- ITP, MIR, WIR, NCR process described?
- Actual Quality Plan or ITP samples included?
- Commissioning protocol?
- Mock-up and sample approval process?

### Risk Management
- Risk register with 15+ risks?
- Risk categories (Technical, Design, Management, HSE, Site, Logistics)?
- Probability/impact matrix?
- Mitigation strategies and owners?
- Or 3 bullet points? → flag as Critical gap

### Document Control
- EDMS described?
- RFI workflow?
- Submittal workflow?
- Version control and distribution matrix?
- Or 2 bullet points? → flag as Critical gap

### Programme
- Gantt chart with milestones?
- S-curve and resource histogram?
- Duration realistic for scope?
- TOC errors ("Error! Bookmark not defined")?

### Team / CVs
- CVs for all key roles?
- Experience relevant to exhibition/museum fit-out?
- Any weak CVs (hospitality/sales background for technical roles)?

### Appendices
- Past Project Portfolio present (3+ projects >2,000 sqm with client, value, scope, duration)?
- Or just a Company Profile appended instead? → flag as Critical gap
- Technical Compliance Matrix?
- RACI Matrix?

## Missing sections to flag

Check for these — if absent, flag as gaps:
- Executive Summary
- Value Engineering
- Stakeholder Communication Plan
- Sustainability (Mostadam/SBC 1001)
- Warranty & After-Sales
- Methodology diagrams/flowcharts

## Scoring rubric

| Score | Meaning |
|-------|---------|
| 9-10 | Submission-ready, minor polish only |
| 7-8 | Good, 2-3 medium fixes needed |
| 5-6 | Adequate, multiple critical gaps |
| <5 | Needs substantial rewrite |

## Common critical gaps (highest severity)

1. Missing Past Project Portfolio appendix (ER requirement)
2. Org chart PM name doesn't match CV section
3. Risk section is 3 bullet points with no register
4. Document Control is 2 bullet points
5. No Executive Summary
6. Exhibition area stated incorrectly or not at all
7. Scope is generic (no gallery names, no designer reference)
8. TOC page numbers not populated

## Output format

Present findings as a table per section with columns: Finding | Severity (Critical/Medium/Low) | Recommended Fix.

End with an Overall Assessment Score and a bullet list of what's missing entirely.

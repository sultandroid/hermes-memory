# DOCX Technical Proposal — Standard Structure

> Reusable section-by-section template for Samaya-branded technical proposals in DOCX format.
> Derived from the RCRC Experience Exhibition proposal (Rev 02, Jul 2026).

## Section Inventory (19 Sections + 6 Appendices)

| # | Section | Content |
|---|---------|---------|
| 1 | Executive Summary | Project overview, key differentiators, key commitments |
| 2 | Company Profile & Qualifications | Entity overview, core competencies, relevant projects, client references |
| 3 | Project Understanding & Approach | Project overview table, gallery descriptions, approach principles |
| 4 | Scope of Work | WBS (10 packages), detailed scope by gallery, supporting areas, exclusions, assumptions |
| 5 | Technical Methodology & Approach | Execution phases (8-phase), methodology flow, method statements per gallery, AV content production |
| 6 | Project Management Plan | Governance (4 levels), project controls, design coordination with designer |
| 7 | Schedule & Time Management | Programme table (12 months), milestones, critical path items |
| 8 | Quality Management | QMS, ITPs by discipline, mock-up approval, NCR procedure, commissioning protocol (8 stages) |
| 9 | Resource & Organisation | Org chart with named personnel, workforce deployment (6 categories), subcontractor management |
| 10 | Health, Safety & Environment | HSE policy/targets, project-specific hazards (8+), environmental management, emergency response |
| 11 | Risk Management | Framework, risk register (15 risks across 6 categories: TEC, DES, MAN, HSE, SIT, LOG) |
| 12 | Procurement & Logistics | Strategy, procurement schedule (16+ items with lead times), shipping, local content |
| 13 | Value Engineering | VE approach, 6+ opportunities with cost-benefit, submission process |
| 14 | Stakeholder Communication Plan | Communication matrix (8 stakeholders), reporting structure, escalation matrix |
| 15 | Sustainability | 7 initiatives with targets, compliance framework (SBC, Mostadam, IKVA) |
| 16 | Warranty & After-Sales Support | Defects liability, manufacturer warranties table, support structure, spare parts |
| 17 | Handover, Training & Operations | 7 deliverables, 5 training modules, 7-step handover process |
| A | Technical Compliance Matrix | 30+ items mapping proposal sections to requirements |
| B | RACI Matrix | 14 activities × 10 roles (including Designer and Client) |
| C | Key Personnel | 16 roles with names, qualifications, experience |
| D | Risk Register | Summary of 15 risks by category with count/severity distribution |
| E | Document Control Procedures | EDMS, transmittal, RFI, submittal workflow, version control, distribution matrix |
| F | Past Project Portfolio | 5+ projects with area, client, scope, duration, completion, key features |

## Risk Register Categories

| Code | Category | Typical Risks |
|------|----------|---------------|
| TEC | Technical | AV integration complexity, LED wall fabrication tolerance, display calibration, show control integration |
| DES | Design | Design development gaps, Stage 3→4 progression, drawing inconsistencies |
| MAN | Management | MEP coordination gap, content production delay, subcontractor performance, warranty demand |
| HSE | Health & Safety | Working at height, electrical safety, Barrisol ceiling heat/fire risk, large-format LED handling |
| SIT | Site | Site access constraints, adjacent area disruption, noise/dust/vibration |
| LOG | Logistics | Long-lead LED delivery, Italian lighting/furniture delay, currency fluctuation |

## Past Project Portfolio — Minimum Fields

Each project entry should include:
- Project Name
- Client
- Location
- Exhibition Area (m2)
- Scope description
- Duration
- Completion year
- Key Features (3-5 bullet points)
